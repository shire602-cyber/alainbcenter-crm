# Git Commit Guidelines - Reduce Vercel Deployments

## Rule: ONE commit per logical change

### When to use `git commit --amend`:
- Fixing syntax errors immediately after a commit
- Fixing typos in the same logical change
- Adding missing pieces to the same feature/fix
- Any follow-up fix within 5 minutes of the original commit

### When to make a NEW commit:
- Different logical change/feature
- Unrelated bug fix
- Different file/component

### Example:
```bash
# BAD - Creates 2 deployments:
git commit -m "Fix auto-reply"
git commit -m "Fix syntax error"  # Should be: git commit --amend --no-edit

# GOOD - Creates 1 deployment:
git commit -m "Fix auto-reply"
# Find syntax error
git add .
git commit --amend --no-edit  # Amends previous commit
git push --force-with-lease
```

### Workflow:
1. Make all related changes
2. Test/build to catch errors
3. Commit once
4. If immediate follow-up needed: `git commit --amend --no-edit`
5. Push with `--force-with-lease` if amended
